# Advanced JavaScript Questions & Answers: 101–200

---

## 101. What is the difference between task queue and microtask queue?

The task queue contains work like timers, user events, and network callbacks. The microtask queue contains promise reactions and `queueMicrotask` callbacks.

```js
setTimeout(() => console.log("task"), 0);

Promise.resolve().then(() => console.log("microtask"));

console.log("sync");
```

Output:

```txt
sync
microtask
task
```

Microtasks run after the current stack clears but before the next task.

---

## 102. Can microtasks starve rendering?

Yes. If microtasks keep scheduling more microtasks, the browser may not get a chance to render.

```js
function loop() {
  queueMicrotask(loop);
}

loop();

setTimeout(() => {
  console.log("This may never run");
}, 0);
```

The microtask queue never becomes empty, so rendering and timers can be blocked.

---

## 103. Why is `await` not exactly the same as `.then()`?

They both use promises, but `await` pauses the async function and resumes it later. `.then()` registers a callback directly.

```js
async function test() {
  console.log("A");
  await Promise.resolve();
  console.log("B");
}

test();
console.log("C");
```

Output:

```txt
A
C
B
```

`await` splits the async function into before-and-after promise continuation parts.

---

## 104. What happens when you `await` a non-promise value?

The value is wrapped like `Promise.resolve(value)`.

```js
async function run() {
  const value = await 42;
  console.log(value);
}

run();

console.log("after");
```

Output:

```txt
after
42
```

Even non-promise values resume asynchronously.

---

## 105. What is promise flattening?

If a promise resolves to another promise, JavaScript adopts the inner promise’s state.

```js
const p = Promise.resolve(
  Promise.resolve("done")
);

p.then(console.log); // "done"
```

The result is not a nested promise value. The outer promise follows the inner promise.

---

## 106. What is a thenable?

A thenable is any object with a callable `then` method. Promises assimilate thenables.

```js
const thenable = {
  then(resolve) {
    resolve("hello");
  }
};

Promise.resolve(thenable).then(console.log); // "hello"
```

This is why promise resolution can interact with non-native promise-like objects.

---

## 107. Why can bad thenables be dangerous?

A thenable can call `resolve` or `reject` manually, throw errors, or behave unexpectedly.

```js
const badThenable = {
  then(resolve, reject) {
    resolve("first");
    reject("second");
    throw new Error("third");
  }
};

Promise.resolve(badThenable).then(console.log); // "first"
```

Promises only settle once, so later attempts are ignored.

---

## 108. What is promise chaining?

Each `.then()` returns a new promise.

```js
Promise.resolve(1)
  .then(x => x + 1)
  .then(x => x * 2)
  .then(console.log); // 4
```

The return value of one handler becomes the input to the next.

---

## 109. What happens if a `.then()` handler throws?

The returned promise becomes rejected.

```js
Promise.resolve()
  .then(() => {
    throw new Error("failed");
  })
  .catch(error => {
    console.log(error.message);
  });
```

Thrown errors inside promise handlers become promise rejections.

---

## 110. What is an unhandled promise rejection?

It happens when a rejected promise has no rejection handler.

```js
Promise.reject(new Error("Boom"));
```

In browsers and Node.js, the host may report this as an unhandled rejection.

Safer:

```js
Promise.reject(new Error("Boom"))
  .catch(error => console.error(error.message));
```

---

## 111. Why should you avoid `new Promise` around existing promises?

This is called the Promise constructor anti-pattern.

```js
// Bad
function getData() {
  return new Promise((resolve, reject) => {
    fetch("/api")
      .then(resolve)
      .catch(reject);
  });
}
```

Better:

```js
function getData() {
  return fetch("/api");
}
```

Wrapping adds complexity and can introduce bugs.

---

## 112. Why does `finally` not receive the resolved value?

`finally` is meant for cleanup, not transforming results.

```js
Promise.resolve("data")
  .finally(() => {
    console.log("cleanup");
  })
  .then(value => {
    console.log(value); // "data"
  });
```

The original value passes through unless `finally` throws or returns a rejected promise.

---

## 113. Can `finally` change promise outcome?

Yes, if it throws or returns a rejected promise.

```js
Promise.resolve("ok")
  .finally(() => {
    throw new Error("cleanup failed");
  })
  .catch(error => console.log(error.message));
```

Output:

```txt
cleanup failed
```

---

## 114. What is async stack trace loss?

Async operations split execution across turns. Older engines sometimes lost the original call chain.

```js
async function a() {
  await b();
}

async function b() {
  throw new Error("fail");
}

a().catch(console.error);
```

Modern engines often preserve async stack traces, but the runtime still executes across separate promise jobs.

---

## 115. What is the difference between concurrency and parallelism?

Concurrency means managing multiple tasks over time. Parallelism means executing multiple tasks at the same time.

```js
async function run() {
  const a = fetch("/a");
  const b = fetch("/b");

  await Promise.all([a, b]);
}
```

This is concurrent. It may or may not be parallel depending on the host and underlying system.

---

## 116. Is JavaScript always single-threaded?

The main JS execution thread is single-threaded in many environments, but JavaScript can use workers, WebAssembly threads, and host-level parallelism.

```js
// main.js
const worker = new Worker("worker.js");

worker.postMessage("start");
```

Worker code runs on a separate thread with isolated memory unless using shared memory.

---

## 117. What is a Web Worker?

A Web Worker runs JavaScript in a background thread separate from the main UI thread.

```js
const worker = new Worker("worker.js");

worker.postMessage({ count: 100 });

worker.onmessage = event => {
  console.log(event.data);
};
```

Workers cannot directly access the DOM.

---

## 118. How do workers communicate?

Workers communicate through message passing.

```js
// main.js
worker.postMessage({ value: 10 });

worker.onmessage = event => {
  console.log("From worker:", event.data);
};
```

```js
// worker.js
self.onmessage = event => {
  self.postMessage(event.data.value * 2);
};
```

Data is cloned or transferred.

---

## 119. What is `postMessage` cloning behavior?

`postMessage` usually uses structured cloning.

```js
worker.postMessage({
  date: new Date(),
  map: new Map([["a", 1]])
});
```

It can preserve more types than JSON, such as `Date`, `Map`, `Set`, and typed arrays.

---

## 120. What is `SharedArrayBuffer`?

`SharedArrayBuffer` allows memory to be shared between threads.

```js
const buffer = new SharedArrayBuffer(4);
const view = new Int32Array(buffer);

view[0] = 123;
```

Unlike normal `ArrayBuffer`, shared memory can be read and written by multiple threads.

---

## 121. Why is `Atomics` needed?

`Atomics` provides safe operations on shared memory.

```js
const buffer = new SharedArrayBuffer(4);
const view = new Int32Array(buffer);

Atomics.store(view, 0, 10);

console.log(Atomics.load(view, 0)); // 10
```

Without atomic operations, concurrent reads and writes can race.

---

## 122. What is a data race?

A data race happens when multiple threads access shared memory and at least one write occurs without proper synchronization.

```js
// Unsafe idea with shared memory:
view[0] = view[0] + 1;
```

Two workers doing this simultaneously can overwrite each other’s updates.

Safer:

```js
Atomics.add(view, 0, 1);
```

---

## 123. What is the difference between cloning and transferring?

Cloning copies data. Transferring moves ownership.

```js
const buffer = new ArrayBuffer(1024);

worker.postMessage(buffer, [buffer]);

console.log(buffer.byteLength); // 0
```

After transfer, the original `ArrayBuffer` is detached.

---

## 124. What is a detached ArrayBuffer?

A detached `ArrayBuffer` no longer owns its memory.

```js
const buffer = new ArrayBuffer(8);

const clone = structuredClone(buffer, {
  transfer: [buffer]
});

console.log(buffer.byteLength); // 0
console.log(clone.byteLength);  // 8
```

Accessing views over detached buffers can fail or behave as empty.

---

## 125. What are typed arrays?

Typed arrays are array-like views over binary buffers.

```js
const buffer = new ArrayBuffer(8);
const view = new Uint8Array(buffer);

view[0] = 255;

console.log(view[0]); // 255
```

They are useful for binary data, files, networking, graphics, and WebAssembly.

---

## 126. What is `DataView`?

`DataView` lets you read and write binary data with control over byte order.

```js
const buffer = new ArrayBuffer(4);
const view = new DataView(buffer);

view.setUint16(0, 256, true);

console.log(view.getUint16(0, true)); // 256
```

The third argument controls little-endian or big-endian interpretation.

---

## 127. What is endianness?

Endianness describes byte order for multi-byte numbers.

```js
const buffer = new ArrayBuffer(2);
const view = new DataView(buffer);

view.setUint16(0, 0x1234, false); // big-endian

console.log(new Uint8Array(buffer)); // [0x12, 0x34]
```

Binary protocols often require explicit endianness.

---

## 128. What is the difference between `ArrayBuffer` and `Blob`?

`ArrayBuffer` is raw binary memory. `Blob` represents immutable file-like binary data.

```js
const blob = new Blob(["hello"], {
  type: "text/plain"
});

const buffer = await blob.arrayBuffer();

console.log(buffer.byteLength);
```

`Blob` is common for files and downloads. `ArrayBuffer` is common for binary processing.

---

## 129. What is the difference between `TextEncoder` and `TextDecoder`?

`TextEncoder` converts strings to bytes. `TextDecoder` converts bytes to strings.

```js
const encoder = new TextEncoder();
const bytes = encoder.encode("hello");

const decoder = new TextDecoder();
console.log(decoder.decode(bytes)); // "hello"
```

---

## 130. What is Unicode normalization?

The same visual character can have different underlying code point sequences.

```js
const a = "\u00E9";      // é
const b = "e\u0301";     // e + accent

console.log(a === b); // false

console.log(a.normalize() === b.normalize()); // true
```

Normalization matters for search, comparison, and user input.

---

## 131. Why can string length be surprising?

JavaScript strings use UTF-16 code units, not user-perceived characters.

```js
console.log("😀".length); // 2
```

The emoji uses a surrogate pair.

Better for user-visible characters:

```js
console.log([... "😀"].length); // 1
```

---

## 132. What are surrogate pairs?

Surrogate pairs represent characters outside the Basic Multilingual Plane using two UTF-16 code units.

```js
const emoji = "😀";

console.log(emoji.length);      // 2
console.log(emoji.codePointAt(0).toString(16));
```

Using `codePointAt` is safer than `charCodeAt` for such characters.

---

## 133. What is the difference between `slice`, `substring`, and `substr`?

`slice` supports negative indexes. `substring` swaps arguments and treats negatives as zero. `substr` is legacy.

```js
const str = "JavaScript";

console.log(str.slice(-6));      // "Script"
console.log(str.substring(4, 0)); // "Java"
console.log(str.substr(4, 6));    // "Script"
```

Prefer `slice`.

---

## 134. Are JavaScript strings mutable?

No. Strings are immutable.

```js
let str = "hello";

str[0] = "H";

console.log(str); // "hello"
```

Operations create new strings.

```js
str = "H" + str.slice(1);
console.log(str); // "Hello"
```

---

## 135. What is boxing of primitives?

When accessing properties on primitives, JavaScript temporarily wraps them in object wrappers.

```js
console.log("hello".toUpperCase()); // "HELLO"
```

Internally, the string primitive behaves as if wrapped by `String`.

But:

```js
const x = "hello";
x.custom = 123;

console.log(x.custom); // undefined
```

The wrapper is temporary.

---

## 136. What is the difference between primitive and object wrappers?

Primitive:

```js
const a = "hello";
```

Object wrapper:

```js
const b = new String("hello");

console.log(typeof a); // "string"
console.log(typeof b); // "object"
console.log(a == b);   // true
console.log(a === b);  // false
```

Avoid wrapper constructors for primitives.

---

## 137. Why is `typeof null` `"object"`?

It is a historical bug in JavaScript.

```js
console.log(typeof null); // "object"
```

Use strict comparison to check for null:

```js
value === null
```

---

## 138. Why is `typeof NaN` `"number"`?

`NaN` is a special numeric value representing invalid numeric results.

```js
console.log(typeof NaN); // "number"
console.log(Number.isNaN(0 / 0)); // true
```

---

## 139. What is negative zero?

JavaScript has both `+0` and `-0`.

```js
console.log(+0 === -0); // true
console.log(Object.is(+0, -0)); // false

console.log(1 / +0); // Infinity
console.log(1 / -0); // -Infinity
```

It matters in some numeric algorithms.

---

## 140. What is BigInt?

`BigInt` represents integers larger than `Number.MAX_SAFE_INTEGER`.

```js
const big = 9007199254740993n;

console.log(big + 1n);
```

You cannot mix `BigInt` and `Number` directly:

```js
// 1n + 1; // TypeError
```

---

## 141. Why do numbers lose precision?

JavaScript numbers are IEEE 754 double-precision floating-point values.

```js
console.log(0.1 + 0.2); // 0.30000000000000004
```

Decimal fractions may not be exactly representable in binary.

---

## 142. What is `Number.EPSILON` used for?

It helps compare floating-point values approximately.

```js
function nearlyEqual(a, b) {
  return Math.abs(a - b) < Number.EPSILON;
}

console.log(nearlyEqual(0.1 + 0.2, 0.3)); // true
```

For larger numbers, use a scaled tolerance.

---

## 143. What is `Number.MAX_SAFE_INTEGER`?

It is the largest integer that can be represented safely.

```js
console.log(Number.MAX_SAFE_INTEGER); // 9007199254740991

console.log(Number.MAX_SAFE_INTEGER + 1);
console.log(Number.MAX_SAFE_INTEGER + 2);
```

Large integers may compare unexpectedly.

---

## 144. What is coercion with objects?

Objects convert to primitives using `Symbol.toPrimitive`, `valueOf`, or `toString`.

```js
const obj = {
  valueOf() {
    return 10;
  }
};

console.log(obj + 5); // 15
```

Custom conversion:

```js
const item = {
  [Symbol.toPrimitive](hint) {
    return hint === "string" ? "item" : 100;
  }
};

console.log(String(item)); // "item"
console.log(item + 1);     // 101
```

---

## 145. What is `Symbol.toPrimitive`?

It defines how an object converts to a primitive.

```js
const money = {
  amount: 50,

  [Symbol.toPrimitive](hint) {
    if (hint === "string") return `$${this.amount}`;
    return this.amount;
  }
};

console.log(+money);        // 50
console.log(`${money}`);    // "$50"
```

---

## 146. Why does `[] + {}` behave strangely?

`+` performs primitive conversion and may concatenate strings.

```js
console.log([] + {}); // "[object Object]"
```

`[]` becomes `""`, `{}` becomes `"[object Object]"`.

But parsing context matters:

```js
// In some contexts, {} may be parsed as a block.
```

---

## 147. Why is `[] == ![]` true?

Because of coercion.

```js
console.log([] == ![]); // true
```

Step idea:

```txt
![]      -> false
[] == false
[] -> ""
false -> 0
"" -> 0
0 == 0 -> true
```

This is why strict equality is preferred.

## 148. What is the difference between `isNaN` and `Number.isNaN`?

Global `isNaN` coerces. `Number.isNaN` does not.

```js
console.log(isNaN("hello"));        // true
console.log(Number.isNaN("hello")); // false

console.log(Number.isNaN(NaN));     // true
```

Prefer `Number.isNaN`.

---

## 149. What is the difference between `parseInt` and `Number`?

`parseInt` parses until it cannot continue. `Number` converts the whole value.

```js
console.log(parseInt("123abc", 10)); // 123
console.log(Number("123abc"));       // NaN
```

Always pass radix to `parseInt`.

```js
parseInt("10", 10);
```

## 150. Why can `toFixed` be surprising?

Floating-point representation affects rounding.

```js
console.log((1.005).toFixed(2)); // often "1.00"
```

The number is not represented exactly as `1.005`.

---

## 151. What is a realm in JavaScript?

A realm is an isolated global environment with its own built-ins.

```js
const iframe = document.createElement("iframe");
document.body.appendChild(iframe);

const arr = new iframe.contentWindow.Array();

console.log(arr instanceof Array); // false
console.log(Array.isArray(arr));   // true
```

`instanceof` can fail across realms.

---

## 152. Why is `Array.isArray` better than `instanceof Array`?

Because `instanceof` depends on the constructor identity.

```js
console.log([] instanceof Array); // true
console.log(Array.isArray([]));   // true
```

Across iframes or realms, `Array.isArray` is more reliable.

---

## 153. How does `instanceof` work?

It checks whether a constructor’s prototype appears in an object’s prototype chain.

```js
function User() {}

const u = new User();

console.log(u instanceof User); // true
```

Roughly:

```js
Object.getPrototypeOf(u) === User.prototype;
```

or somewhere higher in the chain.

---

## 154. Can `instanceof` be customized?

Yes, using `Symbol.hasInstance`.

```js
class Even {
  static [Symbol.hasInstance](value) {
    return typeof value === "number" && value % 2 === 0;
  }
}

console.log(4 instanceof Even); // true
console.log(5 instanceof Even); // false
```

## 155. What is `Object.prototype.toString.call()` useful for?

It gives internal type tags.

```js
console.log(Object.prototype.toString.call([]));       // "[object Array]"
console.log(Object.prototype.toString.call(new Date)); // "[object Date]"
```

Some objects can customize this using `Symbol.toStringTag`.

## 156. What is `Symbol.toStringTag`?

It customizes the label used by `Object.prototype.toString`.

```js
const obj = {
  [Symbol.toStringTag]: "CustomThing"
};

console.log(Object.prototype.toString.call(obj));
// "[object CustomThing]"
```

## 157. What is property shadowing?

An own property can hide a property from the prototype.

```js
const parent = {
  value: 1
};

const child = Object.create(parent);

console.log(child.value); // 1

child.value = 2;

console.log(child.value); // 2
console.log(parent.value); // 1
```

The child’s own property shadows the prototype property.

## 158. What is prototype pollution?

Prototype pollution happens when untrusted input modifies shared prototypes.

```js
const payload = JSON.parse('{"__proto__": {"admin": true}}');

const obj = {};

Object.assign(obj, payload);

console.log({}.admin);
```

In vulnerable patterns, this can affect unrelated objects.

Safer approach:

```js
const safe = Object.create(null);
```

Also validate keys like `__proto__`, `constructor`, and `prototype`.

---

## 159. What is `Object.create(null)`?

It creates an object with no prototype.

```js
const dict = Object.create(null);

dict.key = "value";

console.log(dict.toString); // undefined
```

Useful for dictionary objects because there are no inherited keys.

## 160. What is the difference between `in` and `hasOwnProperty`?

`in` checks own and inherited properties. `hasOwnProperty` checks only own properties.

```js
const parent = { inherited: true };
const child = Object.create(parent);

child.own = true;

console.log("inherited" in child); // true
console.log(child.hasOwnProperty("inherited")); // false
```

Modern safer version:

```js
Object.hasOwn(child, "own");
```

---

## 161. Why use `Object.hasOwn`?

It works even if the object has no prototype or overrides `hasOwnProperty`.

```js
const obj = Object.create(null);

obj.name = "Ava";

console.log(Object.hasOwn(obj, "name")); // true
```

This is safer than:

```js
obj.hasOwnProperty("name");
```

---

## 162. What is the difference between `Object.keys`, `Object.values`, and `Object.entries`?

They return own enumerable string-keyed properties.

```js
const user = {
  name: "Ava",
  age: 30
};

console.log(Object.keys(user));    // ["name", "age"]
console.log(Object.values(user));  // ["Ava", 30]
console.log(Object.entries(user)); // [["name", "Ava"], ["age", 30]]
```

They do not include non-enumerable or symbol properties.

## 163. How do you get symbol properties?

Use `Object.getOwnPropertySymbols` or `Reflect.ownKeys`.

```js
const id = Symbol("id");

const obj = {
  name: "Ava",
  [id]: 123
};

console.log(Object.keys(obj)); // ["name"]
console.log(Object.getOwnPropertySymbols(obj)); // [Symbol(id)]
console.log(Reflect.ownKeys(obj)); // ["name", Symbol(id)]
```

---

## 164. What is `Reflect.ownKeys`?

It returns all own property keys, including string keys and symbol keys, enumerable or not.

```js
const sym = Symbol("secret");

const obj = {};
Object.defineProperty(obj, "hidden", {
  value: 1,
  enumerable: false
});
obj[sym] = 2;

console.log(Reflect.ownKeys(obj)); // ["hidden", Symbol(secret)]
```

## 165. What is the difference between shallow equality and deep equality?

Shallow equality compares top-level references or values. Deep equality recursively compares nested structure.

```js
const a = { nested: { x: 1 } };
const b = { nested: { x: 1 } };

console.log(a === b); // false
console.log(a.nested === b.nested); // false
```

Deep equality would consider them structurally equal, but JavaScript does not provide a built-in general-purpose deep equality function.

---

## 166. What is referential equality?

Objects are equal only if they reference the same object.

```js
const a = {};
const b = {};
const c = a;

console.log(a === b); // false
console.log(a === c); // true
```

This matters heavily in React, memoization, caching, and state management.

---

## 167. What is structural sharing?

Structural sharing creates new objects only along changed paths while reusing unchanged nested objects.

```js
const state = {
  user: { name: "Ava" },
  settings: { theme: "dark" }
};

const nextState = {
  ...state,
  user: {
    ...state.user,
    name: "Mia"
  }
};

console.log(nextState.settings === state.settings); // true
console.log(nextState.user === state.user);         // false
```

This improves performance and enables cheap change detection.

## 168. What is memoization?

Memoization caches function results based on inputs.

```js
function memoize(fn) {
  const cache = new Map();

  return function (arg) {
    if (cache.has(arg)) {
      return cache.get(arg);
    }

    const result = fn(arg);
    cache.set(arg, result);
    return result;
  };
}

const square = memoize(x => x * x);

console.log(square(4)); // computes
console.log(square(4)); // cached
```

Be careful: unbounded caches can cause memory leaks.

---

## 169. How can WeakMap help memoization?

WeakMap avoids keeping object keys alive.

```js
function memoizeObject(fn) {
  const cache = new WeakMap();

  return function (obj) {
    if (cache.has(obj)) {
      return cache.get(obj);
    }

    const result = fn(obj);
    cache.set(obj, result);
    return result;
  };
}
```

When the object key becomes unreachable, the cached entry can be garbage collected.

---

## 170. What is a memory leak caused by an unbounded Map?

A `Map` strongly holds keys and values.

```js
const cache = new Map();

function remember(user) {
  cache.set(user.id, user);
}
```

If entries are never removed, memory usage grows forever.

Possible fix:

```js
function remember(user) {
  cache.set(user.id, user);

  if (cache.size > 1000) {
    const firstKey = cache.keys().next().value;
    cache.delete(firstKey);
  }
}
```

## 171. What is LRU cache?

LRU means Least Recently Used. It evicts the oldest unused item.

```js
class LRUCache {
  constructor(limit) {
    this.limit = limit;
    this.map = new Map();
  }

  get(key) {
    if (!this.map.has(key)) return undefined;

    const value = this.map.get(key);
    this.map.delete(key);
    this.map.set(key, value);

    return value;
  }

  set(key, value) {
    if (this.map.has(key)) {
      this.map.delete(key);
    }

    this.map.set(key, value);

    if (this.map.size > this.limit) {
      const oldestKey = this.map.keys().next().value;
      this.map.delete(oldestKey);
    }
  }
}
```

`Map` preserves insertion order, which makes simple LRU implementations possible.

## 172. What is debounce?

Debounce delays execution until calls stop for a period.

```js
function debounce(fn, delay) {
  let timerId;

  return function (...args) {
    clearTimeout(timerId);

    timerId = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

const onSearch = debounce(query => {
  console.log("Searching", query);
}, 300);
```

Useful for search boxes and resize events.

---

## 173. What is throttle?

Throttle limits execution to at most once per interval.

```js
function throttle(fn, interval) {
  let lastTime = 0;

  return function (...args) {
    const now = Date.now();

    if (now - lastTime >= interval) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}

const onScroll = throttle(() => {
  console.log("scroll");
}, 200);
```

Useful for scroll and mousemove events.

---

## 174. What is the difference between debounce and throttle?

Debounce waits until activity stops. Throttle runs at controlled intervals during activity.

```js
const debouncedResize = debounce(() => {
  console.log("resize ended");
}, 300);

const throttledScroll = throttle(() => {
  console.log("scrolling");
}, 300);
```

Debounce is best for final action. Throttle is best for continuous feedback.

---

## 175. What is function currying?

Currying transforms a function with multiple arguments into a chain of unary functions.

```js
function add(a) {
  return function (b) {
    return a + b;
  };
}

console.log(add(2)(3)); // 5
```

Useful for partial application and function composition.

---

## 176. What is partial application?

Partial application fixes some arguments of a function.

```js
function multiply(a, b) {
  return a * b;
}

function partial(fn, ...fixedArgs) {
  return function (...remainingArgs) {
    return fn(...fixedArgs, ...remainingArgs);
  };
}

const double = partial(multiply, 2);

console.log(double(5)); // 10
```

Currying returns nested functions one argument at a time. Partial application pre-fills some arguments.

---

## 177. What is function composition?

Composition combines functions so the output of one becomes input to another.

```js
const compose = (f, g) => x => f(g(x));

const double = x => x * 2;
const increment = x => x + 1;

const result = compose(double, increment);

console.log(result(3)); // 8
```

`increment(3)` gives `4`, then `double(4)` gives `8`.

---

## 178. What is a pure function?

A pure function returns the same output for the same input and has no side effects.

```js
function add(a, b) {
  return a + b;
}
```

Impure:

```js
let total = 0;

function addToTotal(x) {
  total += x;
}
```

Pure functions are easier to test and reason about.

---

## 179. What is a side effect?

A side effect is any interaction with outside state.

```js
function saveUser(user) {
  localStorage.setItem("user", JSON.stringify(user));
}
```

Examples include modifying globals, writing to storage, logging, DOM updates, network requests, and mutation of external objects.

---

## 180. What is idempotency?

An operation is idempotent if repeating it has the same effect as doing it once.

```js
const user = {
  active: false
};

function deactivate(user) {
  user.active = false;
}

deactivate(user);
deactivate(user);
```

The final state is the same after one or many calls.

---

## 181. What is reentrancy?

A function is reentrant if it can be safely called again before a previous call finishes.

```js
let locked = false;

function notReentrant() {
  if (locked) {
    throw new Error("Already running");
  }

  locked = true;

  // work

  locked = false;
}
```

Reentrancy matters with callbacks, event handlers, and async code.

---

## 182. What is mutation during iteration?

Changing a collection while iterating can produce surprising results.

```js
const arr = [1, 2, 3];

for (const value of arr) {
  console.log(value);

  if (value === 2) {
    arr.push(4);
  }
}
```

Depending on iteration type, newly added items may also be visited.

Safer:

```js
for (const value of [...arr]) {
  // iterate over snapshot
}
```

---

## 183. How does `for...in` differ from `for...of`?

`for...in` iterates enumerable property keys. `for...of` iterates iterable values.

```js
const arr = ["a", "b"];

for (const key in arr) {
  console.log(key); // "0", "1"
}

for (const value of arr) {
  console.log(value); // "a", "b"
}
```

Use `for...of` for arrays.

---

## 184. What is the difference between array methods and loops?

Array methods communicate intent but may allocate intermediate arrays.

```js
const result = [1, 2, 3, 4]
  .filter(x => x % 2 === 0)
  .map(x => x * 2);
```

For hot performance-sensitive paths, a loop may avoid intermediate allocations.

```js
const result = [];

for (const x of [1, 2, 3, 4]) {
  if (x % 2 === 0) {
    result.push(x * 2);
  }
}
```

---

## 185. What is lazy iteration?

Lazy iteration computes values only when requested.

```js
function* map(iterable, fn) {
  for (const item of iterable) {
    yield fn(item);
  }
}

const values = map([1, 2, 3], x => x * 2);

console.log(values.next().value); // 2
```

Generators are useful for lazy pipelines.

---

## 186. What are async iterators?

Async iterators produce values over time.

```js
async function* fetchPages() {
  yield await Promise.resolve("page 1");
  yield await Promise.resolve("page 2");
}

for await (const page of fetchPages()) {
  console.log(page);
}
```

They are useful for streams, paginated APIs, and event-like data.

---

## 187. What is `for await...of`?

It consumes async iterables.

```js
async function* numbers() {
  yield 1;
  yield 2;
}

async function run() {
  for await (const n of numbers()) {
    console.log(n);
  }
}

run();
```

Each iteration waits for the next async result.

---

## 188. What is backpressure?

Backpressure means slowing the producer when the consumer cannot keep up.

```js
const stream = new ReadableStream({
  pull(controller) {
    controller.enqueue("chunk");
  }
});
```

In streams, backpressure prevents unbounded buffering and memory growth.

---

## 189. What is a ReadableStream?

A `ReadableStream` represents a source of data that can be consumed in chunks.

```js
const response = await fetch("/large-file");

const reader = response.body.getReader();

while (true) {
  const { value, done } = await reader.read();

  if (done) break;

  console.log("chunk", value);
}
```

This avoids loading the entire response into memory at once.

---

## 190. What is a TransformStream?

A `TransformStream` transforms data passing through it.

```js
const upperCaseStream = new TransformStream({
  transform(chunk, controller) {
    controller.enqueue(chunk.toUpperCase());
  }
});
```

It has a writable side and a readable side.

---

## 191. What is the DOM?

The DOM is a tree representation of a document exposed to JavaScript by the browser.

```js
const title = document.querySelector("h1");

title.textContent = "Hello";
```

The DOM is not part of ECMAScript. It is provided by the browser host environment.

---

## 192. What is layout thrashing?

Layout thrashing happens when code repeatedly reads layout and writes layout, forcing the browser to recalculate styles/layout many times.

```js
const boxes = document.querySelectorAll(".box");

for (const box of boxes) {
  const height = box.offsetHeight; // read
  box.style.height = height + 10 + "px"; // write
}
```

Better batch reads and writes:

```js
const heights = [...boxes].map(box => box.offsetHeight);

boxes.forEach((box, i) => {
  box.style.height = heights[i] + 10 + "px";
});
```

---

## 193. What is repaint vs reflow?

Reflow recalculates layout. Repaint updates pixels without necessarily changing layout.

```js
element.style.width = "200px"; // likely reflow
element.style.color = "red";   // likely repaint
```

Layout changes are usually more expensive than paint-only changes.

---

## 194. Why is `transform` often better than `top/left` for animations?

`transform` can often be handled by the compositor without recalculating layout.

```js
box.style.transform = "translateX(100px)";
```

Changing layout properties:

```js
box.style.left = "100px";
```

`left` may trigger layout, while `transform` is usually more animation-friendly.

---

## 195. What is `requestAnimationFrame`?

`requestAnimationFrame` schedules work before the browser’s next repaint.

```js
function animate() {
  box.style.transform = `translateX(${Date.now() % 500}px)`;

  requestAnimationFrame(animate);
}

requestAnimationFrame(animate);
```

It is preferred for visual updates over `setInterval`.

---

## 196. What is `requestIdleCallback`?

`requestIdleCallback` schedules low-priority work when the browser is idle.

```js
requestIdleCallback(deadline => {
  while (deadline.timeRemaining() > 0) {
    // do small background work
    break;
  }
});
```

It is useful for non-urgent tasks, though availability varies by environment.

---

## 197. What is event delegation?

Event delegation attaches one listener to a parent instead of many listeners to children.

```js
document.querySelector("#list").addEventListener("click", event => {
  if (event.target.matches("button")) {
    console.log("Button clicked:", event.target.textContent);
  }
});
```

This works because events bubble up the DOM tree.

---

## 198. What is event bubbling?

Event bubbling means an event starts at the target and propagates upward through ancestors.

```js
child.addEventListener("click", () => {
  console.log("child");
});

parent.addEventListener("click", () => {
  console.log("parent");
});
```

Clicking `child` logs:

```txt
child
parent
```

---

## 199. What is event capturing?

Capturing is the phase where an event travels from the root down to the target.

```js
parent.addEventListener(
  "click",
  () => {
    console.log("parent capture");
  },
  true
);

child.addEventListener("click", () => {
  console.log("child bubble");
});
```

With capture enabled, the parent listener runs before the child bubble listener.

---

## 200. What is the full DOM event flow?

The DOM event flow has three phases:

1. Capturing phase
2. Target phase
3. Bubbling phase

```js
parent.addEventListener(
  "click",
  () => console.log("capture"),
  true
);

child.addEventListener("click", () => console.log("target"));

parent.addEventListener("click", () => console.log("bubble"));
```

Clicking `child` outputs:

```txt
capture
target
bubble
```

A senior-level answer connects this to delegation, propagation control, listener options, memory leaks, and browser rendering behavior.
