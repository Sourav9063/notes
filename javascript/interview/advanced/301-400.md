# Advanced JavaScript Questions & Answers: 301–400

---

## 301. What is an ECMAScript specification algorithm?

The ECMAScript spec describes behavior using abstract algorithms, not implementation code.

```js
const result = [3, 1, 2].sort();
console.log(result); // ["1", "2", "3"] style ordering for strings
```

A senior answer explains that the spec defines observable behavior, while engines choose their own internal implementation.

---

## 302. What is an abstract operation?

An abstract operation is a spec-level helper used to define language behavior.

Example: when JavaScript compares values, it may internally use operations like `ToPrimitive`, `ToNumber`, or `SameValue`.

```js
console.log("5" - 1); // 4
```

This involves converting `"5"` to a number.

---

## 303. What is `ToPrimitive`?

`ToPrimitive` converts an object into a primitive value.

```js
const obj = {
  valueOf() {
    return 10;
  }
};

console.log(obj + 5); // 15
```

The object becomes `10` before addition.

---

## 304. What is `ToBoolean`?

`ToBoolean` converts a value to `true` or `false`.

```js
console.log(Boolean(0));        // false
console.log(Boolean(""));       // false
console.log(Boolean([]));       // true
console.log(Boolean({}));       // true
```

Objects are always truthy, even empty arrays and empty objects.

---

## 305. What is `ToNumber`?

`ToNumber` converts a value into a number.

```js
console.log(Number("42"));   // 42
console.log(Number(""));     // 0
console.log(Number(null));   // 0
console.log(Number(true));   // 1
console.log(Number(false));  // 0
console.log(Number(undefined)); // NaN
```

Many surprising coercions come from `ToNumber`.

---

## 306. What is `ToString`?

`ToString` converts a value to a string.

```js
console.log(String(123));       // "123"
console.log String(true);       // SyntaxError
```

Correct version:

```js
console.log(String(true));      // "true"
console.log(String(null));      // "null"
console.log(String(undefined)); // "undefined"
```

Objects usually use `toString`.

```js
console.log(String({})); // "[object Object]"
```

---

## 307. What is `SameValueZero`?

`SameValueZero` is an equality comparison used by `Set`, `Map`, and `Array.prototype.includes`.

```js
console.log([NaN].includes(NaN)); // true

const set = new Set();
set.add(NaN);
set.add(NaN);

console.log(set.size); // 1
```

It treats `NaN` as equal to `NaN`.

---

## 308. Why does `indexOf` not find `NaN`?

`indexOf` uses strict equality, where `NaN !== NaN`.

```js
const arr = [NaN];

console.log(arr.indexOf(NaN));  // -1
console.log(arr.includes(NaN)); // true
```

Use `includes` when `NaN` matters.

---

## 309. What is `SameValue`?

`SameValue` is similar to `Object.is`.

```js
console.log(Object.is(NaN, NaN)); // true
console.log(Object.is(+0, -0));   // false
```

This differs from `===`.

```js
console.log(NaN === NaN); // false
console.log(+0 === -0);   // true
```

---

## 310. What is `OrdinaryOwnPropertyKeys`?

It defines the order of own property keys.

```js
const obj = {
  b: 1,
  2: "two",
  a: 2,
  1: "one"
};

console.log(Object.keys(obj)); // ["1", "2", "b", "a"]
```

Integer-like keys come first in ascending order, then string keys in insertion order, then symbols.

---

## 311. Why is object property order not simply insertion order?

Integer-like keys are ordered numerically before normal string keys.

```js
const obj = {};

obj.foo = "first";
obj[2] = "second";
obj[1] = "third";

console.log(Object.keys(obj)); // ["1", "2", "foo"]
```

This matters when using objects as ordered maps.

---

## 312. Why is `Map` better for ordered key-value data?

`Map` preserves insertion order for all key types.

```js
const map = new Map();

map.set("foo", "first");
map.set(2, "second");
map.set(1, "third");

console.log([...map.keys()]); // ["foo", 2, 1]
```

Objects have special ordering rules for integer-like keys.

---

## 313. What is an array exotic object?

Arrays are special objects with custom behavior for `length` and numeric indexes.

```js
const arr = [];

arr[2] = "x";

console.log(arr.length); // 3
console.log(arr);        // [empty × 2, "x"]
```

Setting an index can update `length`.

---

## 314. What happens when you reduce array length?

Elements beyond the new length are removed.

```js
const arr = [1, 2, 3, 4];

arr.length = 2;

console.log(arr); // [1, 2]
```

This is not just metadata; it deletes elements.

---

## 315. Can array length assignment fail?

Yes, if elements are non-configurable.

```js
const arr = [1, 2, 3];

Object.defineProperty(arr, "2", {
  configurable: false
});

arr.length = 1;

console.log(arr.length); // may remain greater than 1
```

The engine cannot delete a non-configurable element.

---

## 316. What is a sparse array?

A sparse array has missing indexes.

```js
const arr = [];

arr[3] = "x";

console.log(arr.length); // 4
console.log(0 in arr);   // false
```

Sparse arrays behave differently from arrays containing `undefined`.

---

## 317. What is the difference between a hole and `undefined`?

A hole means the property does not exist. `undefined` is an actual value.

```js
const a = [undefined];
const b = [,];

console.log(0 in a); // true
console.log(0 in b); // false
```

Some array methods skip holes.

---

## 318. Which array methods skip holes?

Methods like `map`, `forEach`, `filter`, and `some` skip holes.

```js
const arr = [1, , 3];

arr.forEach(x => console.log(x));
```

Output:

```txt
1
3
```

But `for...of` sees holes as `undefined`.

```js
for (const x of arr) {
  console.log(x);
}
```

Output:

```txt
1
undefined
3
```

---

## 319. What does `Array.from` do with holes?

`Array.from` converts holes into `undefined`.

```js
const arr = [1, , 3];

const copy = Array.from(arr);

console.log(copy);       // [1, undefined, 3]
console.log(1 in copy);  // true
```

This can change sparse arrays into dense arrays.

---

## 320. What does spread do with holes?

Array spread also turns holes into `undefined`.

```js
const arr = [1, , 3];

const copy = [...arr];

console.log(copy);      // [1, undefined, 3]
console.log(1 in copy); // true
```

---

## 321. What is `Array.prototype.sort` default behavior?

By default, `sort` converts elements to strings and sorts lexicographically.

```js
const nums = [10, 2, 1];

nums.sort();

console.log(nums); // [1, 10, 2]
```

For numeric sort:

```js
nums.sort((a, b) => a - b);
console.log(nums); // [1, 2, 10]
```

---

## 322. Is JavaScript sort stable?

Modern JavaScript requires stable sorting for `Array.prototype.sort`.

```js
const users = [
  { name: "A", score: 10 },
  { name: "B", score: 10 },
  { name: "C", score: 5 }
];

users.sort((a, b) => a.score - b.score);

console.log(users.map(u => u.name)); // ["C", "A", "B"]
```

Equal items keep their relative order.

---

## 323. Why must sort comparator be consistent?

An inconsistent comparator can produce unpredictable ordering.

```js
const arr = [1, 2, 3, 4];

arr.sort(() => Math.random() - 0.5);
```

This is not a reliable shuffle.

Better shuffle:

```js
function shuffle(arr) {
  const copy = [...arr];

  for (let i = copy.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [copy[i], copy[j]] = [copy[j], copy[i]];
  }

  return copy;
}
```

---

## 324. What is a comparator contract?

A comparator should be deterministic, antisymmetric, and transitive.

```js
items.sort((a, b) => a.age - b.age);
```

This means:

```txt
a before b -> negative
a after b  -> positive
equal      -> zero
```

Breaking this can confuse sort algorithms.

## 325. What is `Intl.Collator`?

`Intl.Collator` compares strings according to locale rules.

```js
const collator = new Intl.Collator("en", {
  sensitivity: "base"
});

console.log(collator.compare("a", "A")); // 0
```

Useful for sorting human-readable text.

---

## 326. Why is locale string sorting different from simple sorting?

Simple sorting compares Unicode code units. Locale sorting follows language rules.

```js
const names = ["ä", "z", "a"];

console.log([...names].sort());

const collator = new Intl.Collator("de");
console.log([...names].sort(collator.compare));
```

Locale-aware sorting is better for user-facing data.

## 327. What is `Intl.NumberFormat`?

It formats numbers for locales.

```js
const formatter = new Intl.NumberFormat("en-US", {
  style: "currency",
  currency: "USD"
});

console.log(formatter.format(1234.5)); // "$1,234.50"
```

Avoid hand-building localized number strings.

---

## 328. What is `Intl.DateTimeFormat`?

It formats dates according to locale and timezone.

```js
const formatter = new Intl.DateTimeFormat("en-US", {
  dateStyle: "full",
  timeStyle: "short",
  timeZone: "Asia/Dhaka"
});

console.log(formatter.format(new Date()));
```

This avoids many timezone display bugs.

---

## 329. Why is `Date` difficult in JavaScript?

`Date` mixes timestamp storage with local and UTC accessors.

```js
const date = new Date("2026-05-11");

console.log(date.toISOString());
console.log(date.toString());
```

The same timestamp displays differently depending on timezone.

---

## 330. What is wrong with parsing date strings manually?

Different formats can be ambiguous.

```js
new Date("05/11/2026"); // May 11 or November 5 depending on assumptions?
```

Prefer ISO-like formats and explicit timezone handling.

```js
new Date("2026-05-11T00:00:00Z");
```

---

## 331. What is a timestamp?

A timestamp is usually milliseconds since Unix epoch in JavaScript.

```js
const now = Date.now();

console.log(now); // milliseconds since 1970-01-01T00:00:00Z
```

`Date` stores an absolute point in time internally.

---

## 332. What is timezone offset?

Timezone offset is the difference between local time and UTC.

```js
const date = new Date();

console.log(date.getTimezoneOffset());
```

The value is in minutes and can change due to daylight saving time.

---

## 333. Why is daylight saving time hard?

Some local times do not exist or happen twice.

```js
const date = new Date(2026, 2, 8, 2, 30);
console.log(date.toString());
```

In some timezones, 2:30 AM during DST transition may be skipped or shifted.

---

## 334. What is monotonic time?

Monotonic time only moves forward and is better for measuring durations.

```js
const start = performance.now();

// work

const end = performance.now();

console.log(end - start);
```

`Date.now()` can be affected by system clock changes.

---

## 335. Why use `performance.now()` instead of `Date.now()` for benchmarking?

`performance.now()` has higher precision and is monotonic.

```js
const start = performance.now();

for (let i = 0; i < 1_000_000; i++) {}

console.log(performance.now() - start);
```

Use it for measuring elapsed time.

---

## 336. What is the Performance API?

It provides tools for measuring browser performance.

```js
performance.mark("start");

// work

performance.mark("end");
performance.measure("work", "start", "end");

console.log(performance.getEntriesByName("work"));
```

Useful for real instrumentation.

---

## 337. What is `PerformanceObserver`?

It observes performance entries.

```js
const observer = new PerformanceObserver(list => {
  for (const entry of list.getEntries()) {
    console.log(entry.name, entry.duration);
  }
});

observer.observe({ entryTypes: ["measure"] });
```

This helps collect metrics without polling.

---

## 338. What is long task detection?

Long tasks are blocks of work that keep the main thread busy for too long.

```js
const observer = new PerformanceObserver(list => {
  for (const entry of list.getEntries()) {
    console.log("Long task:", entry.duration);
  }
});

observer.observe({ entryTypes: ["longtask"] });
```

Long tasks can make UIs feel frozen.

---

## 339. What is First Input Delay conceptually?

It measures delay between a user’s first interaction and when the browser can respond.

```js
button.addEventListener("click", () => {
  console.log("click handled");
});
```

If the main thread is blocked, the click waits.

---

## 340. How do you reduce main-thread blocking?

Break large work into chunks.

```js
async function process(items) {
  for (let i = 0; i < items.length; i++) {
    doWork(items[i]);

    if (i % 100 === 0) {
      await new Promise(resolve => setTimeout(resolve, 0));
    }
  }
}
```

This gives the event loop chances to process input and rendering.

---

## 341. What is cooperative scheduling?

Cooperative scheduling means long-running code voluntarily yields.

```js
async function work(items) {
  for (const item of items) {
    processItem(item);

    await Promise.resolve();
  }
}
```

However, yielding to microtasks only may still block rendering. For UI, yield to tasks or animation frames.

---

## 342. Why is yielding with microtasks not enough for rendering?

Microtasks run before rendering. A long chain of microtasks can still block painting.

```js
async function badYield(items) {
  for (const item of items) {
    processItem(item);
    await Promise.resolve();
  }
}
```

Better:

```js
function nextFrame() {
  return new Promise(resolve => requestAnimationFrame(resolve));
}

async function betterYield(items) {
  for (const item of items) {
    processItem(item);
    await nextFrame();
  }
}
```

---

## 343. What is `requestAnimationFrame` timing?

It runs before the browser paints the next frame.

```js
requestAnimationFrame(timestamp => {
  console.log("Before paint:", timestamp);
});
```

Use it for visual updates.

---

## 344. Why should DOM reads and writes be batched?

Interleaving reads and writes can force repeated layout recalculations.

```js
// Bad
for (const el of elements) {
  const h = el.offsetHeight;
  el.style.height = h + 10 + "px";
}
```

Better:

```js
const heights = elements.map(el => el.offsetHeight);

elements.forEach((el, i) => {
  el.style.height = heights[i] + 10 + "px";
});
```

---

## 345. What is layout invalidation?

Changing layout-affecting styles marks layout as dirty.

```js
box.style.width = "300px";

console.log(box.offsetWidth); // forces layout calculation
```

Reading layout after writing can force synchronous reflow.

---

## 346. What is compositing?

Compositing combines layers into the final screen image.

```js
box.style.transform = "translateX(20px)";
box.style.opacity = "0.5";
```

Some properties can be animated on the compositor, avoiding layout and paint work.

---

## 347. What is `will-change`?

`will-change` hints that an element will change.

```css
.card {
  will-change: transform;
}
```

Use it sparingly. Overusing it can waste memory by creating too many layers.

---

## 348. What is image lazy loading?

It delays image loading until the image is near the viewport.

```html
<img src="large.jpg" loading="lazy" alt="Large image">
```

This reduces initial page load cost.

---

## 349. What is preloading?

Preloading tells the browser to fetch important resources early.

```html
<link rel="preload" href="/font.woff2" as="font" crossorigin>
```

Use for critical resources needed soon.

---

## 350. What is prefetching?

Prefetching fetches resources likely needed later.

```html
<link rel="prefetch" href="/next-page.js">
```

It is lower priority than preload and should not be used for critical current-page assets.

---

## 351. What is module preloading?

It preloads JavaScript modules and their dependencies.

```html
<link rel="modulepreload" href="/app.js">
```

Useful for ESM-based browser apps.

---

## 352. What is resource priority?

Browsers assign priorities to resources like scripts, CSS, images, and fonts.

```html
<img src="hero.jpg" fetchpriority="high" alt="Hero image">
```

Poor priority choices can delay important rendering resources.

---

## 353. What is render-blocking CSS?

CSS blocks rendering because the browser needs styles before painting.

```html
<link rel="stylesheet" href="/styles.css">
```

Large CSS files can delay first render.

---

## 354. Why are synchronous scripts parser-blocking?

Classic scripts block HTML parsing while they download and execute.

```html
<script src="/app.js"></script>
```

The browser pauses parsing because the script might modify the document.

---

## 355. What is the difference between `defer` and `async`?

`defer` downloads in parallel and executes after HTML parsing, in order.

```html
<script src="/app.js" defer></script>
```

`async` downloads in parallel and executes as soon as ready, not necessarily in order.

```html
<script src="/analytics.js" async></script>
```

---

## 356. How do module scripts behave by default?

Module scripts are deferred by default.

```html
<script type="module" src="/app.js"></script>
```

They do not block HTML parsing like classic synchronous scripts.

---

## 357. What is import map?

An import map controls how module specifiers resolve in browsers.

```html
<script type="importmap">
{
  "imports": {
    "utils": "/src/utils.js"
  }
}
</script>

<script type="module">
  import { helper } from "utils";
</script>
```

This allows bare-style imports in browsers.

---

## 358. What is dynamic import chunking?

Bundlers often turn dynamic imports into separate chunks.

```js
const module = await import("./admin-panel.js");
module.render();
```

The admin panel code can be loaded only when needed.

---

## 359. What is the waterfall problem?

A waterfall happens when resources load sequentially instead of in parallel.

```js
const a = await fetch("/a");
const b = await fetch("/b");
const c = await fetch("/c");
```

Better:

```js
const [a, b, c] = await Promise.all([
  fetch("/a"),
  fetch("/b"),
  fetch("/c")
]);
```

---

## 360. What is over-fetching?

Over-fetching means requesting more data than needed.

```js
const user = await fetch("/user-with-all-posts-comments-likes");
```

Better:

```js
const user = await fetch("/user-summary");
```

Over-fetching wastes bandwidth, memory, and parsing time.

---

## 361. What is under-fetching?

Under-fetching means needing many follow-up requests because the first response lacks required data.

```js
const user = await fetch("/user/1").then(r => r.json());
const posts = await fetch(`/users/${user.id}/posts`).then(r => r.json());
```

Sometimes batching or parallel fetching helps.

```js
const [user, posts] = await Promise.all([
  fetch("/user/1").then(r => r.json()),
  fetch("/users/1/posts").then(r => r.json())
]);
```

---

## 362. What is request deduplication?

It avoids sending the same request multiple times.

```js
const pending = new Map();

function fetchOnce(url) {
  if (pending.has(url)) return pending.get(url);

  const promise = fetch(url).finally(() => pending.delete(url));
  pending.set(url, promise);

  return promise;
}
```

Useful in UI apps where multiple components request the same data.

---

## 363. What is stale cache invalidation?

It decides when cached data is too old to trust.

```js
const cache = new Map();

function getCached(key, maxAgeMs) {
  const entry = cache.get(key);

  if (!entry) return null;

  if (Date.now() - entry.time > maxAgeMs) {
    cache.delete(key);
    return null;
  }

  return entry.value;
}
```

Caching is easy; invalidation is the hard part.

---

## 364. What is optimistic UI?

Optimistic UI updates the interface before the server confirms success.

```js
async function likePost(post) {
  post.liked = true;

  try {
    await fetch(`/posts/${post.id}/like`, { method: "POST" });
  } catch {
    post.liked = false;
  }
}
```

It improves perceived speed but needs rollback logic.

---

## 365. What is pessimistic UI?

Pessimistic UI waits for the server before updating.

```js
async function likePost(post) {
  await fetch(`/posts/${post.id}/like`, { method: "POST" });

  post.liked = true;
}
```

It is safer but may feel slower.

---

## 366. What is retry with exponential backoff?

It retries failed operations with increasing delays.

```js
async function retry(fn, attempts = 3) {
  for (let i = 0; i < attempts; i++) {
    try {
      return await fn();
    } catch (error) {
      if (i === attempts - 1) throw error;

      const delay = 2 ** i * 100;
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}
```

Useful for transient network failures.

---

## 367. What is jitter in retry logic?

Jitter adds randomness to avoid many clients retrying at the same time.

```js
const baseDelay = 1000;
const jitter = Math.random() * 500;

await new Promise(resolve => {
  setTimeout(resolve, baseDelay + jitter);
});
```

This prevents retry storms.

---

## 368. What is request timeout with `AbortController`?

You can abort a request after a time limit.

```js
async function fetchWithTimeout(url, ms) {
  const controller = new AbortController();

  const id = setTimeout(() => controller.abort(), ms);

  try {
    return await fetch(url, {
      signal: controller.signal
    });
  } finally {
    clearTimeout(id);
  }
}
```

This prevents hanging requests.

---

## 369. What is idempotent HTTP behavior relevant to JS clients?

An idempotent request can be safely repeated without changing the final state beyond the first request.

```js
await fetch("/user/123", {
  method: "PUT",
  body: JSON.stringify({ name: "Ava" })
});
```

`PUT` is typically idempotent. `POST` often is not.

---

## 370. What is an idempotency key?

An idempotency key lets a server recognize duplicate requests.

```js
await fetch("/payments", {
  method: "POST",
  headers: {
    "Idempotency-Key": crypto.randomUUID(),
    "Content-Type": "application/json"
  },
  body: JSON.stringify({ amount: 100 })
});
```

Useful for payment and order creation flows.

---

## 371. What is `crypto.randomUUID()`?

It creates a random UUID string.

```js
const id = crypto.randomUUID();

console.log(id);
```

Useful for client-side request IDs, not for security tokens by itself.

---

## 372. What is `crypto.getRandomValues()`?

It fills typed arrays with cryptographically strong random values.

```js
const bytes = new Uint8Array(16);

crypto.getRandomValues(bytes);

console.log(bytes);
```

Use this instead of `Math.random()` for security-sensitive randomness.

---

## 373. Why is `Math.random()` not secure?

`Math.random()` is not designed for cryptographic security.

```js
const token = Math.random().toString(36); // bad for secrets
```

Use Web Crypto:

```js
const bytes = new Uint8Array(32);
crypto.getRandomValues(bytes);
```

---

## 374. What is SubtleCrypto?

`crypto.subtle` provides cryptographic operations like hashing, signing, and encryption.

```js
const data = new TextEncoder().encode("hello");

const hash = await crypto.subtle.digest("SHA-256", data);

console.log(new Uint8Array(hash));
```

It is asynchronous and promise-based.

---

## 375. Why should you not invent your own crypto?

Cryptography is easy to get subtly wrong.

```js
function fakeEncrypt(text) {
  return btoa(text);
}
```

Base64 is encoding, not encryption. Use standard, reviewed crypto APIs and protocols.

---

## 376. What is constant-time comparison?

It avoids leaking information through timing differences.

```js
function insecureCompare(a, b) {
  return a === b;
}
```

For server-side secrets, use platform-specific timing-safe comparison APIs when available.

In Node:

```js
const crypto = require("crypto");

crypto.timingSafeEqual(
  Buffer.from(a),
  Buffer.from(b)
);
```

## 377. What is prototype poisoning?

Prototype poisoning modifies prototypes to affect object behavior.

```js
Object.prototype.isAdmin = true;

const user = {};

console.log(user.isAdmin); // true
```

Avoid trusting inherited properties.

```js
Object.hasOwn(user, "isAdmin");
```

---

## 378. Why should you avoid merging untrusted objects blindly?

Attackers may use keys like `__proto__`, `constructor`, or `prototype`.

```js
function merge(target, source) {
  for (const key in source) {
    target[key] = source[key];
  }
}
```

Safer:

```js
function safeMerge(target, source) {
  for (const key of Object.keys(source)) {
    if (["__proto__", "constructor", "prototype"].includes(key)) {
      continue;
    }

    target[key] = source[key];
  }
}
```

## 379. What is sandboxing?

Sandboxing isolates code to limit what it can access.

```js
const iframe = document.createElement("iframe");

iframe.sandbox = "allow-scripts";
iframe.src = "/untrusted.html";

document.body.appendChild(iframe);
```

Sandboxing is useful when embedding untrusted content.

## 380. Why is `eval` dangerous?

`eval` executes strings as code.

```js
const userInput = "alert(document.cookie)";

eval(userInput); // dangerous
```

It creates security risks and can prevent optimization.

---

## 381. Is `new Function()` safer than `eval`?

Not generally. It still executes string code.

```js
const fn = new Function("return document.cookie");

console.log(fn());
```

It has different scoping behavior than `eval`, but it is still dangerous with untrusted input.

---

## 382. What is direct vs indirect `eval`?

Direct `eval` can access local scope. Indirect `eval` executes in global scope.

```js
function test() {
  const secret = 123;

  eval("console.log(secret)"); // 123

  const indirect = eval;
  indirect("console.log(typeof secret)"); // "undefined"
}

test();
```

Direct `eval` is especially harmful for optimization and security.

---

## 383. What is CSP nonce?

A nonce allows specific inline scripts under Content Security Policy.

```html
<script nonce="random-value">
  console.log("allowed");
</script>
```

The server must generate a unique nonce and include it in the CSP header.

---

## 384. What is Trusted Types?

Trusted Types helps prevent DOM XSS by restricting dangerous DOM sinks like `innerHTML`.

```js
// Conceptual example
const policy = trustedTypes.createPolicy("default", {
  createHTML(input) {
    return sanitize(input);
  }
});

element.innerHTML = policy.createHTML(userInput);
```

It forces unsafe HTML creation through controlled policies.

---

## 385. What is sanitization?

Sanitization removes dangerous parts of untrusted input.

```js
// Bad
element.innerHTML = userInput;
```

Better:

```js
element.textContent = userInput;
```

When HTML is truly needed, use a trusted sanitizer rather than regex.

---

## 386. Why is regex-based HTML sanitization unsafe?

HTML parsing is complex, and regex usually misses edge cases.

```js
function unsafeSanitize(html) {
  return html.replace(/<script.*?>.*?<\/script>/gi, "");
}
```

This does not handle many dangerous payloads. Use a proper HTML sanitizer.

---

## 387. What is JSON injection?

JSON injection happens when untrusted content is inserted into JSON or script contexts unsafely.

```html
<script>
  window.DATA = {"name": "</script><script>alert(1)</script>"}
</script>
```

Escape data correctly when embedding JSON in HTML.

---

## 388. Why should JSON be served with the correct content type?

Serving JSON as JavaScript can create security issues.

```txt
Content-Type: application/json
```

This helps browsers treat the response as data, not executable script.

---

## 389. What is dependency confusion?

Dependency confusion happens when a package manager installs a malicious public package instead of an intended private package.

```json
{
  "dependencies": {
    "@company/internal-utils": "1.0.0"
  }
}
```

Mitigate with scoped registries, lockfiles, and package manager configuration.

---

## 390. Why are lockfiles important?

Lockfiles pin exact dependency versions.

```txt
package-lock.json
pnpm-lock.yaml
yarn.lock
```

They make installs more reproducible and reduce accidental supply-chain changes.

---

## 391. What is semantic versioning?

Semantic versioning uses:

```txt
MAJOR.MINOR.PATCH
```

Example:

```json
{
  "dependencies": {
    "lib": "^1.2.3"
  }
}
```

`^1.2.3` may install newer compatible versions like `1.5.0`, but not `2.0.0`.

---

## 392. What is the difference between `^` and `~` in package versions?

`^` allows compatible minor and patch updates. `~` allows patch updates.

```json
{
  "dependencies": {
    "a": "^1.2.3",
    "b": "~1.2.3"
  }
}
```

Possible ranges:

```txt
^1.2.3 -> 1.x.x, below 2.0.0
~1.2.3 -> 1.2.x, below 1.3.0
```

---

## 393. What is a peer dependency?

A peer dependency says: “The consuming project must provide this dependency.”

```json
{
  "peerDependencies": {
    "react": "^18.0.0"
  }
}
```

Libraries use this to avoid bundling duplicate framework copies.

---

## 394. What is a transitive dependency?

A transitive dependency is a dependency of a dependency.

```txt
your-app
  -> package-a
      -> package-b
```

Security and compatibility issues can come from transitive dependencies too.

---

## 395. What is dependency hoisting?

Package managers may move shared dependencies higher in `node_modules`.

```txt
node_modules/
  lodash/
  package-a/
  package-b/
```

Hoisting can reduce duplication but may expose undeclared dependencies accidentally.

---

## 396. What is the phantom dependency problem?

A phantom dependency is used by your code but not declared in your package.

```js
import leftPad from "left-pad";
```

If `left-pad` exists only because another package installed it transitively, your app may break later.

Declare direct dependencies explicitly.

---

## 397. What is ESM/CJS dual package hazard?

A package exposing both ESM and CommonJS can accidentally create two separate module instances.

```js
// One part imports ESM
import pkg from "some-package";

// Another part requires CJS
const pkg2 = require("some-package");
```

State may be duplicated if the package resolves to different files.

---

## 398. What is package entry resolution?

Package entry resolution decides which file loads for an import.

```json
{
  "main": "./dist/index.cjs",
  "module": "./dist/index.js",
  "exports": {
    ".": {
      "import": "./dist/index.js",
      "require": "./dist/index.cjs"
    }
  }
}
```

Modern tools usually prefer `exports`.

---

## 399. What is the difference between runtime dependency and dev dependency?

Runtime dependencies are needed by the app/library when it runs. Dev dependencies are needed for development, build, or testing.

```json
{
  "dependencies": {
    "express": "^4.18.0"
  },
  "devDependencies": {
    "vitest": "^1.0.0"
  }
}
```

Shipping dev dependencies to production can increase install size and attack surface.

---

## 400. What is senior-level JavaScript understanding at this stage?

It means understanding not just syntax, but how the language, engine, browser, Node.js runtime, package system, network, security model, and performance model interact.

```js
async function loadDashboard() {
  performance.mark("dashboard:start");

  const controller = new AbortController();

  const [user, stats] = await Promise.all([
    fetch("/api/user", { signal: controller.signal }).then(r => r.json()),
    fetch("/api/stats", { signal: controller.signal }).then(r => r.json())
  ]);

  performance.mark("dashboard:data");

  renderDashboard({ user, stats });

  performance.mark("dashboard:end");
  performance.measure(
    "dashboard:data-load",
    "dashboard:start",
    "dashboard:data"
  );
  performance.measure(
    "dashboard:render",
    "dashboard:data",
    "dashboard:end"
  );
}
```

A beginner sees `async/await`.

A stronger developer sees concurrent fetches.

A senior developer sees cancellation, network waterfalls, rendering cost, measurement points, error handling gaps, cache strategy, user experience, and runtime scheduling.
